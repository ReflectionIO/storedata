Opens Sans is our title font, used for all headings, labels, buttons and calls to action.

Lato is used as body copy in all blocks of text, table data and additionally in tabs and navigation menus where all caps links are required.

Body Copy

Body copy is all Lato Regular with a colour of #5b5b68. There are 3 main sizes based on application and size of container (see right for sizes and line-heights).

Feature body copy is used sparingly in short bursts as an introduction or to draw attention to important content.

Main body copy is the most widely used and should be the default when creating text content. However if the containing element is small or narrow or the content is less important then Minor body copy must be used to maintain legibility and content hierarchy.

Bolded text is Lato Bold and coloured #29292c for extra prominence.

Blockquotes

Blockquotes are Open Sans Semi-bold and #5b5b68. When the large font size is used on a dark coloured background however the font must be Open Sans Bold. The quote mark is always a slightly darker shade of the background colour except on dark grey-black backgrounds when it becomes a slightly lighter shade. The font used for this speech mark is Source Sans Pro.

The size and colour of quote used depends on the application but as a general rule the large quotes are used in more prominent places such as a blog article or a welcome banner. Smaller quotes are used in narrow columns or slide out panels or in areas where the quote is not as prominent.

The Code

Font CSS classes and SASS variables

Font classes are available for explicit styling of indiviual HTML elements, but it is preferable to style site components using SASS partials, one partial per component, to keep CSS styles encapsulated per component. Extending classes in SASS reduces the amount of CSS output in the main stylesheet, but for our solution component CSS should be kept separate from global CSS for integration purposes.

For styles that will appear in the global stylesheet, @extend the font classes in the SASS where a particular font is required for an element.

For site components use the available SASS variables for font family styling.


								/* Available font classes */
								.open-sans
								.open-sans-semi
								.open-sans-bold
								.lato
								.lato-bold

								/* Extend a font class in the SASS if the selector is to be included in the main stylesheet */
								@extend .open-sans;

								/* Font variables for use in site components/SASS partials */
								$brand-body-font: 'Lato', sans-serif;
								$brand-heading-font: 'Open Sans', sans-serif;

								/* Example: */
								font-family: $brand-body-font;
								font-weight: 700;

Headings

Use one h1 heading per page for SEO purposes.

Components should use headings in the correct semantic order and each heading has base font styles, including font family, size, line height and top/bottom margins. Where a heading is required to be styled using another heading's styles, use the available heading classes.

For instance, an h2 should be followed by an h3, but if the h3 is to have the styles of an h4, the h3 can be given the class "heading-style--heading-four". This is demonstrated below.

Font size units

To generate a font size in ems from the SASS call the mixin font-size() with the px value required. This will output a font size in ems in the CSS.

Font sizes are stated in ems and inherit from the body px font size. For more control over a particular component font size at different breakpoints, the container of the component should have it's own explicit font size. This should be in px if the component is to remain unaffected by changes in body font size. Children of the component will inherit from the fixed component font size and not respond to changes in body font size.

The body font size changes at specific breakpoints to increase the font size of all inheriting elements.


								/* Font size mixin for calculating ems
								@include font-size(14px);

Paragraphs

Default styling is in place for the <p> element. Use SASS partials to style <p> elements specific to components if necessary.

For featured paragraphs use the class "p-style--featured" on the <p> tag or on the container of the element.

For minor paragraphs use the class "p-style--minor" on the <p> tag or (preferably) on the container of the element. For instance, for smaller width columns use the "p-style--minor" on the column container to ensure all paragraphs have the "minor" styling.


								/* Classes for paragraph styles
								.p-style--minor
								.p-style--featured

Ordered and unordered lists

Ordered and unordered lists in content require classes, because to provide styles based on just the <ol> or <ul> tag alone would mean that all lists on the site would inherit those styles, which is not suitable for lists that are not text content.

For a list with default font sizing and styles use the class "content-list" on the <ol> or <ul> tag.

For a minor list, use the class "content-list--minor".

For a featured list, use the class "content-list--featured".


								/* Classes for ordered and unordered lists
								ul.content-list
								ul.content-list--minor
								ul.content-list--featured
								ol.content-list
								ol.content-list--minor
								ol.content-list--featured
