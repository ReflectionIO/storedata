Universal Loading Bar

Any action performed on the site which results in a short period of loading or processing initiates the universal pop-up loading bar. This bar is subtle enough to not be too distracting but visible enough via it’s animation effects to keep the user informed of what’s happening on the site, and where possible, how long it will take. The bar rises up from the bottom of the browser window and overlays all of the page content. It is slightly transparent and has a loading message contained within which can be changed depending on what is processing.

There are 2 versions: a default animated striped version when the loading time is not known and a solid version which fills as loading progresses (see demos).

Once complete the page content will refresh or reload with the relevant transition or animation and the bar slides out of view.

Component Loading Bar

Where the loading or processing relates to a specific content item (such as a graph or a pop-up box) the loading bar appears within the actual element itself. See the demos on the right to see a couple of examples.

The loading bars themselves have the same properties as the universal loading bar and behave in the same manner.

Loading Buttons

Loading buttons are used in any application where pressing of a button initiates some kind of processing (e.g. submit a form, post a comment, download a file, save changes etc).

As with the universal loading bar they indicate clearly through animation that an action is being processed and (where possible) how long that action will take.

Ellipsis Loader

When specific items on a page take a short time to appear the ellipsis loader is used. It is very subtle so as not to be too prominent but must indicate to the user that a value or other item is about to appear in that location once loaded/processed. It avoids the problem of having empty gaps all over a page as data is loaded in the background.

It is used in place of items such as app titles or data values in tables (see examples on the right).
